## US-1 Запуск и регистрация

**Как** новый пользователь,  
**я хочу** отправить `/start` и сразу попасть в главное меню,  
**чтобы** начать пользоваться ботом без лишних шагов.

### Сценарий: первый запуск
1. Пользователь отправляет `/start`.
2. Бот ищет `telegram_id` в таблице `users`.
3. Запись не найдена → создаёт нового пользователя со значениями по умолчанию.
4. Отправляет приветственное сообщение и показывает главное меню.

### Сценарий: повторный запуск
1. Пользователь отправляет `/start`.
2. Бот находит существующую запись.
3. Показывает главное меню без приветствия.

### Примечания
- Если в момент `/start` было активно отслеживание — оно останавливается, все сообщения-ордера удаляются.
- Поля пользователя при создании: `telegram_id`, `username`, `created_at`, `last_active_at`.

## US-2 Главное меню

**Как** пользователь,  
**я хочу** видеть главное меню с доступными действиями,  
**чтобы** быстро перейти к нужному разделу.

### Сценарий
1. Бот показывает одно сообщение с кнопками:
   - `📋 Мои фильтры` — всегда активна.
   - `⚙️ Настройки` — всегда активна.

### Примечания
- Главное меню достижимо из любого раздела кнопкой `← Главное меню` или командой `/start`.

## US-3 Просмотр списка фильтров

**Как** пользователь,  
**я хочу** видеть все свои фильтры, сгруппированные по паре (валюта + тип сделки),  
**чтобы** выбрать нужный для запуска или редактирования.

### Сценарий: нет фильтров
1. Нажимаю `📋 Мои фильтры`.
2. Бот показывает сообщение: «У вас пока нет фильтров».
3. Кнопки: `➕ Создать фильтр`, `← Главное меню`.

### Сценарий: есть фильтры
1. Нажимаю `📋 Мои фильтры`.
2. Бот отправляет **одно сообщение на каждый фильтр** (не все в одном сообщении).
3. Каждое сообщение содержит:
   - Название фильтра, пара, тип сделки.
   - Краткое резюме ключевых параметров (диапазон суммы, мин. репутация, кол-во ордеров).
   - Кнопки: `▶️ Старт`, `✏️ Редактировать`, `🗑 Удалить`.
4. После всех сообщений-фильтров — одно общее сообщение с кнопками `➕ Создать фильтр`, `← Главное меню`.

### Примечания
- При навигации обратно в этот раздел предыдущие сообщения-фильтры удаляются, отправляются новые (актуальные данные).
- Группировка по парам отображается только в тексте каждого сообщения, не отдельными заголовками.

## US-4 Создание фильтра

**Как** пользователь,  
**я хочу** создать именованный фильтр с набором условий для выбранной валютной пары,  
**чтобы** видеть только те ордера, которые мне подходят.

### Сценарий: пошаговый wizard

**Шаг 1 — Выбор фиатной валюты**
- Бот показывает кнопки с валютами: `RUB`, `USD`, `EUR`, `KZT`, `UAH`, `GBP`, `TRY`, `AZN`, `UZS` (и другие из ответа Bybit API).
- Кнопка `❌ Отмена`.

**Шаг 2 — Тип сделки**
- Бот показывает: `📈 Покупка USDT` / `📉 Продажа USDT`.
- Кнопки `← Назад`, `❌ Отмена`.

**Шаг 3 — Название фильтра**
- Бот просит ввести название текстом (например: «Утренний», «Крупные сделки»).
- Максимум 32 символа.
- Кнопки `← Назад`, `❌ Отмена`.

**Шаг 4 — Настройка параметров**
- Бот открывает экран параметров фильтра (см. US-5).
- Все поля необязательны — можно сохранить фильтр без условий.

### Параметры фильтра

| Параметр | Описание | По умолчанию |
|---|---|---|
| Мин. сумма сделки | Показывать ордера, у которых `maxAmount ≥ значение` (USDT) | не задано |
| Макс. сумма сделки | Показывать ордера, у которых `minAmount ≤ значение` (USDT) | не задано |
| Мин. курс | Нижняя граница цены в фиате | не задано |
| Макс. курс | Верхняя граница цены в фиате | не задано |
| Мин. число сделок продавца | Скрывать продавцов с меньшим числом завершённых сделок | не задано |
| Мин. completion rate | Скрывать продавцов с репутацией ниже N% | не задано |
| Ордера без описания | Показывать или скрывать | показывать |
| Белый список слов | Показывать только ордера, где описание содержит хотя бы одно слово | пусто |
| Чёрный список слов | Скрывать ордера, где описание содержит хотя бы одно слово | пусто |
| Сортировка | По курсу: возрастание или убывание | возрастание |
| Кол-во ордеров | Сколько ордеров отображать (1–5) | 5 |

### Фильтрация по диапазону суммы
Диапазоны пересекаются: показываем ордера, у которых `minAmount ≤ наш_макс` И `maxAmount ≥ наш_мин`.

### Примечания
- Одна пара (например, USDT/RUB Покупка) может иметь несколько именованных фильтров.
- При нажатии `❌ Отмена` на любом шаге — черновик удаляется, пользователь возвращается в список фильтров.

## US-5 Редактирование фильтра

**Как** пользователь,  
**я хочу** изменить параметры существующего фильтра,  
**чтобы** скорректировать условия, не удаляя фильтр.

### Сценарий
1. В списке фильтров нажимаю `✏️ Редактировать` у нужного фильтра.
2. Бот показывает экран параметров с текущими значениями.
3. Каждый параметр — отдельная кнопка. При нажатии — бот запрашивает новое значение текстом (или переключает toggle).
4. После изменения возвращается к экрану параметров с обновлённым значением.
5. Нажимаю `✅ Сохранить` — фильтр обновлён, возврат в список фильтров.

### Подэкраны редактирования

**Диапазон суммы / Диапазон курса:** два последовательных шага (мин → макс), каждый можно пропустить кнопкой `⏭ Пропустить`.

**Опыт и репутация:** два поля на одном экране — `✏️ Мин. число сделок` и `✏️ Мин. Completion Rate`.

**Фильтр по описанию:**
- Toggle `📝 Без описания: показывать ↔ скрывать` — переключается кнопкой без ввода текста.
- `✏️ Белый список слов` — ввод слов через запятую, полностью заменяет список. Кнопка `🗑 Очистить`.
- `✏️ Чёрный список слов` — аналогично.

**Сортировка и количество:**
- Выбор направления кнопками: `↑ По курсу: возрастание` / `↓ По курсу: убывание`.
- Изменение количества — ввод числа от 1 до 5.

## US-6 Удаление фильтра

**Как** пользователь,  
**я хочу** удалить ненужный фильтр с подтверждением,  
**чтобы** случайно не потерять нужный.

### Сценарий
1. В списке фильтров нажимаю `🗑 Удалить`.
2. Бот запрашивает подтверждение: «Удалить фильтр "Утренний"? Это действие нельзя отменить.»
3. Кнопки: `✅ Да, удалить`, `❌ Отмена`.
4. При подтверждении — фильтр удалён из БД, сообщение этого фильтра удаляется из чата, остальные сообщения-фильтры остаются.

### Примечания
- Если фильтр был активен в момент удаления — отслеживание немедленно останавливается.

## US-7 Запуск отслеживания

**Как** пользователь,  
**я хочу** запустить отслеживание по выбранному фильтру,  
**чтобы** видеть актуальные ордера, подходящие под мои условия.

### Сценарий
1. В списке фильтров нажимаю `▶️ Старт` у нужного фильтра.
2. Все сообщения списка фильтров удаляются.
3. Бот делает первый запрос к Bybit API с параметрами фильтра.
4. API возвращает список ордеров, к которым применяются клиентские фильтры (диапазон суммы, курс, репутация, ключевые слова, персональный чёрный список описаний).
5. Бот загружает в буфер **первые 15 ордеров** после фильтрации (или меньше, если подходящих меньше).
6. Бот отправляет:
   - **1 сообщение-заголовок** (статус, название фильтра, время, кол-во найденных).
   - **До 5 сообщений-ордеров** (из буфера).
7. Запускается цикл обновления каждые 5 секунд.

### Примечания
- В один момент времени у пользователя может быть активным только одно отслеживание.
- Запуск нового отслеживания автоматически останавливает предыдущее (если было активно).

## US-8 Активное отслеживание — отображение ордеров

**Как** пользователь при активном отслеживании,  
**я хочу** видеть актуальный список ордеров, обновляемый каждые 5 секунд,  
**чтобы** не пропустить выгодное предложение.

### Структура сообщений в чате

```
[ Сообщение-заголовок ]      ← редактируется каждые 5 сек
[ Сообщение: ордер #1 ]      ← удаляется и создаётся заново
[ Сообщение: ордер #2 ]      ← удаляется и создаётся заново
[ Сообщение: ордер #3 ]      ← удаляется и создаётся заново
[ Сообщение: ордер #4 ]      ← удаляется и создаётся заново
[ Сообщение: ордер #5 ]      ← удаляется и создаётся заново
```

### Сообщение-заголовок (редактируется)

```
🟢 Отслеживание активно
USDT/RUB • 📈 Покупка • "Утренний"
🕐 14:32:05  •  Найдено: 47
```
Кнопки: `[⏹ Остановить отслеживание]`

### Сообщение-ордер

```
#1  •  92.50 RUB/USDT
👤 TraderPro  ⭐ 98.5%  (1 240 сд.)
💰 500 – 50 000 USDT
📝 "Работаю круглосуточно, СБП"
```
Кнопки: `[❌ Не подходит]  [💚 Купить →]`

- Если описание отсутствует — поле `📝` не отображается.
- `💚 Купить →` — `url_button`, ведёт напрямую на страницу ордера на Bybit.

### Цикл обновления (каждые 5 секунд)

1. Бот запрашивает свежие данные у Bybit API.
2. Применяет клиентские фильтры + персональный чёрный список описаний.
3. Обновляет буфер: берёт первые 15 подходящих ордеров.
4. Через очередь (с небольшой задержкой между операциями для соблюдения лимитов Telegram):
   - Удаляет все текущие сообщения-ордера.
   - Отправляет новые сообщения-ордера (из первых N буфера, где N ≤ 5).
5. Редактирует сообщение-заголовок (обновляет время и счётчик найденных).

### Сценарий: нет подходящих ордеров

Сообщения-ордера отсутствуют. Заголовок показывает:

```
🟢 Отслеживание активно
USDT/RUB • 📈 Покупка • "Утренний"
🕐 14:32:05

😔 По вашему фильтру ордеров не найдено.
Ожидание подходящих предложений...
```

### Сценарий: ошибка API

Сообщения-ордера не обновляются. Заголовок показывает:

```
⚠️ Не удалось получить данные от Bybit.
Повторная попытка через 5 сек...
```

## US-9 Отметить ордер как неподходящий

**Как** пользователь при активном отслеживании,  
**я хочу** скрыть ордер с конкретным описанием навсегда,  
**чтобы** этот и похожие ордера больше не появлялись в списке.

### Сценарий
1. Нажимаю `❌ Не подходит` у нужного ордера.
2. Бот сохраняет `sha256(description.strip().lower())` в персональный чёрный список описаний пользователя в БД.
3. Сообщение этого ордера немедленно удаляется из чата.
4. Бот берёт следующий ордер из буфера (6-й по счёту) и немедленно отправляет новое сообщение-ордер.
5. Если буфер исчерпан — новое сообщение не отправляется; место остаётся пустым до следующего цикла обновления.
6. Всплывающее уведомление (`answerCallbackQuery`): «✅ Ордер скрыт. Описание добавлено в чёрный список.»

### Примечания
- Хранится хэш описания, а не сам текст — для экономии места и унификации.
- Все ордера с совпадающим хэшем скрываются во всех фильтрах этого пользователя.
- Таймер автостопа (5 минут) сбрасывается при каждом нажатии этой кнопки.

## US-10 Перейти к ордеру на Bybit

**Как** пользователь при активном отслеживании,  
**я хочу** нажать «Купить» и открыть страницу ордера на Bybit,  
**чтобы** совершить сделку напрямую на бирже.

### Сценарий
1. Нажимаю `💚 Купить →` у нужного ордера.
2. Telegram открывает in-app браузер с URL страницы ордера на Bybit.
3. Сообщения бота остаются без изменений; отслеживание продолжается.

### Примечания
- `💚 Купить →` реализован как `InlineKeyboardButton` с `url=` (не callback).
- Таймер автостопа (5 минут) **не сбрасывается** этой кнопкой (она не генерирует callback).
- URL формируется на основе `ad_id` из ответа Bybit API.

## US-11 Остановка отслеживания вручную

**Как** пользователь при активном отслеживании,  
**я хочу** остановить отслеживание нажатием кнопки,  
**чтобы** вернуться к управлению фильтрами.

### Сценарий
1. Нажимаю `⏹ Остановить отслеживание` в сообщении-заголовке.
2. Цикл обновления останавливается.
3. Все сообщения-ордера удаляются из чата (через очередь).
4. Сообщение-заголовок редактируется:

```
⏹ Отслеживание остановлено.
USDT/RUB • 📈 Покупка • "Утренний"
```
Кнопки: `[▶️ Возобновить]  [← К фильтрам]`

### Сценарий: навигация прочь
- Если пользователь нажимает `← К фильтрам` или `/start` при активном отслеживании — отслеживание останавливается автоматически, сообщения-ордера удаляются.

## US-12 Автостоп по неактивности

**Как** пользователь,  
**я хочу**, чтобы отслеживание автоматически останавливалось после 5 минут без активности,  
**чтобы** не нагружать систему вхолостую.

### Сценарий
1. Таймер (5 минут) запускается при старте отслеживания и сбрасывается при каждом нажатии `❌ Не подходит`.
2. Кнопка `💚 Купить →` таймер **не сбрасывает** (нет callback).
3. По истечении 5 минут:
   - Цикл обновления останавливается.
   - Все сообщения-ордера удаляются.
   - Сообщение-заголовок редактируется:

```
⏸ Отслеживание остановлено автоматически
(5 минут без активности).
USDT/RUB • 📈 Покупка • "Утренний"
```
Кнопки: `[▶️ Возобновить]  [← К фильтрам]`

### Примечания
- «Возобновить» запускает новый цикл отслеживания с тем же фильтром, буфер перезагружается.

## US-13 Управление чёрным списком описаний

**Как** пользователь,  
**я хочу** просматривать и очищать персональный чёрный список описаний ордеров,  
**чтобы** вернуть ранее скрытые ордера.

### Сценарий: список пуст
1. `⚙️ Настройки` → `🚫 Чёрный список описаний`.
2. Бот: «Список пуст. Нажмите "Не подходит" у ордера, чтобы добавить его сюда.»
3. Кнопка `← Назад`.

### Сценарий: есть записи
1. Бот отправляет **одно сообщение на каждую запись** чёрного списка.
2. Каждое сообщение содержит:
   - Порядковый номер.
   - Текст описания ордера (оригинальный, сохранённый при добавлении).
   - Кнопка `🗑 Удалить`.
3. После всех сообщений — общее сообщение с кнопками `🗑 Очистить всё` и `← Назад`.

### Сценарий: удаление записи
1. Нажимаю `🗑 Удалить` у нужной записи.
2. Запись удаляется из БД, сообщение этой записи удаляется из чата.

### Сценарий: очистить всё
1. Нажимаю `🗑 Очистить всё`.
2. Бот запрашивает подтверждение: «Удалить все записи чёрного списка?»
3. Кнопки: `✅ Да, очистить`, `❌ Отмена`.
4. При подтверждении — все записи удалены, все сообщения-записи удалены из чата.

### Примечания
- В БД хранится как хэш (`sha256`), так и оригинальный текст описания (для отображения пользователю).
- Удаление записи из чёрного списка не возвращает ордер мгновенно — он появится на следующем цикле обновления.
